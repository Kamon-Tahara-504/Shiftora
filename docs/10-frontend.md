# 10. フロントエンド

## 技術

- **Next.js**（SPA。SSR は不要。）
- 理由: React エコシステム、API 接続が楽、SaaS UI と相性がいい、ポートフォリオとして評価されやすい。

---

## 画面一覧

| ロール | 画面数 | 画面 |
|--------|--------|------|
| **org_admin** | 3 | 職員管理 / シフト生成 / シフトカレンダー |
| **staff** | 2 | 自分のシフト / 希望休 |

---

## 画面境界（実装コストに直結するので割り切る）

**職員管理:** 一覧・追加・編集・**無効化**。**削除は物理削除しない**（シフト履歴が壊れる）。`employees.is_active` で無効化する。

**シフト生成 UI:** 「Generate Shift」ボタン→API を呼ぶ→数秒待つ。**非同期キューは不要**。OR-Tools でも月次なら数秒で終わる。解なしのときは **missing_slots をそのまま画面に表示**する。

**シフトカレンダー:** 閲覧＋手動修正（PATCH）を呼べる。
